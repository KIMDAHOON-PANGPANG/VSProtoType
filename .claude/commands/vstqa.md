VSQA 스킬을 사용해 **TQA**를 수행하라.

- 인자가 없으면: 최신 `build/index_N.html` 기준으로 기본 배치(기준 시드)를 돌리고, 직전 로그가 있으면 회귀 비교까지 한다.
- 진단 플래그(지배빌드·죽은무기·사망클러스터·밀도·적체)를 반드시 보고에 포함하고, 로그를 `QA/TQA/bot/logs/`에 저장한다.
- 배치가 끝나면 `.claude/skills/VSQA/assets/tqa-리포트-template.html` 템플릿으로 HTML 리포트를 만들어
  `QA/TQA/bot/logs/`에 저장하고 링크를 안내한다(사용자가 체크+의견 → `/vstqa 후속 요청 …` 복사).
- 사용자가 `/vstqa 후속 요청 …` 을 붙여넣으면: 체크된 항목별 조치안을 제안하고 진행한다.
- 스윕 요청(특정 수치 실험)이면 노브 1~2개만, 값당 3시드로 돌리고 "값 → 지표 변화 → 해석" 표로 보고한다.
