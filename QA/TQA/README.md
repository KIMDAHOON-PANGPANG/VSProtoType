# TQA — Technical QA (스펙·수치 검증)

**"스펙대로 도는가, 수치가 목표 범위에 있는가"** 를 기계(봇)가 검증하는 영역.
재미 판정은 하지 않는다 — 그건 [FQA](../FQA/README.md) 의 몫.

## 구성

| 경로 | 역할 |
|---|---|
| `bot/` | 밸런스 QA 봇 시뮬레이터 — 게임을 iframe 으로 띄워 자동 플레이·가속·지표 수집 ([사용법](bot/README.md)) |
| `bot/logs/` | 배치 런 로그·리포트 누적 (날짜-내용 명명) |

## TQA 가 하는 일

1. **스펙 검증** — `Base/implemented_spec.html` 기준으로 시스템이 명세대로 동작하는지
   (예: 처형 대상 수 = maxTargets, 그로기 2.4초, 전염 변이 최대 3기).
2. **밸런스 계측** — 시나리오 봇 배치로 생존/딜지분/분당 곡선/처형 지표 수집.
3. **자동 진단** — 지배 빌드 · 죽은 무기 · 사망 클러스터 · 밀도 · 파워커브 적체 플래그.
4. **회귀 체크** — 버전업 후 같은 시드 배치를 돌려 이전 로그와 비교 (시드 결정적).
5. **FQA 지원** — [FQA 재미 기준](../FQA/FUN_CRITERIA.md)이 지표로 번역되면,
   그 목표 범위를 TQA 배치로 측정해 판정 시트에 공급한다.

## 기준 문서

- 스펙: `Base/implemented_spec.html` (항상 최신 인덱스 기준으로 갱신)
- 밸런스 단일 소스: `build/gameBalance.json`
- 리서치 근거: `Research/`
