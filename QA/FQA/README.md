# FQA — Fun QA (재미 판정 루프)

> 실행은 **VSQA 스킬**(`.claude/skills/VSQA`)이 담당한다 — "FQA 세션 돌려줘"라고만 하면 된다.

**"재밌는가"** 를 사람(기획자)이 판정하고, Claude 가 그 판정을 기준으로 학습해
밸런스를 다시 맞추는 영역. 수치 검증은 [TQA](../TQA/README.md) 가 맡는다.

> 핵심 원칙: **재미의 판정자는 항상 사람.** Claude 는 ① 판정하기 좋은 재료를 만들고
> ② 판정을 기준(FUN_CRITERIA)으로 축적하고 ③ 기준에 맞게 밸런스를 조정할 뿐이다.

## 구성

| 경로 | 역할 |
|---|---|
| `FUN_CRITERIA.md` | **재미 기준 원장 (단일 소스)** — 판정이 쌓여 만들어지는 기준 표 |
| `sessions/` | 세션별 판정 시트 — `NNN_YYYY-MM-DD_주제/` 폴더로 누적 |
| `sessions/_TEMPLATE_판정시트.md` | 판정 시트 템플릿 |

## 협업 루프 (이렇게 일한다)

```
① 사용자: "FQA 세션 돌려줘" (그냥 이렇게만, 또는 + 이번에 보고 싶은 것)
      ↓
② Claude: TQA 봇 배치 실행 (+ 필요하면 밸런스 변형 A/B안)
      → sessions/NNN_날짜_주제/판정시트.md 생성
      → 시트에는 "판단 요소"가 사람 언어로 정리됨
         (예: "2분 러시 때 HP가 40%까지 밀렸다 — 이 정도 쫄림이 맞나?")
      → 각 요소마다 실플레이로 확인할 포인트 안내 ("2분 구간만 플레이해보세요")
      ↓
③ 사용자: build/index_N.html 실플레이 + 시트의 판정란에 기입
      ⭕ 재밌다 / ❌ 재미없다 / ➖ 모르겠다 + 이유 한 줄 (거칠게 적어도 됨)
      ※ 시트에 안 적고 채팅으로 불러줘도 됨 — Claude 가 시트에 대신 기입
      ↓
④ Claude (재학습): 판정을 FUN_CRITERIA.md 의 기준으로 승격·갱신
      → 사람 언어 기준을 봇 지표로 번역 (예: "쫄리는 맛" → 받은피해/분 8~20 + 빈사 1~2회)
      ↓
⑤ Claude (재조정): 기준 미달 항목의 밸런스를 튜닝 (gameBalance.json / 새 index_N)
      → TQA 봇으로 "기준 범위 안에 들어왔는지" 수치 검증 후 새 판 제시
      ↓
⑥ ①로 — 사용자가 다시 판정. 기준이 쌓일수록 Claude 의 첫 제안이 정확해진다.
```

## 판정 팁 (사용자용)

- **기준은 문장 하나면 충분** — "처형이 너무 자주라 특별하지 않음" 같은 한 줄이면
  Claude 가 지표(처형/분 상한)로 번역해서 원장에 올린다.
- **❌ 가 더 값지다** — 재미없는 순간의 이유가 기준을 만든다.
- **➖ 도 판정이다** — 판단이 안 서는 항목은 Claude 가 A/B 변형을 만들어 다음 세션에서 비교시켜 준다.
- 기준이 서로 충돌하면(예: 압박↑ vs 성장 쾌감↑) Claude 가 충돌을 표시하고 우선순위를 물어본다.

## FQA 와 TQA 의 분업

| | TQA | FQA |
|---|---|---|
| 질문 | 스펙대로 도는가 · 수치가 범위인가 | 재밌는가 |
| 판정자 | 봇 (자동) | **사람** |
| 산출물 | 배치 로그·진단 플래그 | 판정 시트 → 재미 기준 원장 |
| 주기 | 버전업마다 회귀 배치 | 사용자가 원할 때 세션 |
