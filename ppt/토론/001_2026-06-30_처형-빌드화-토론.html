처형 플레이를 빌드로 키울 수 있을까

플레이어가 직접 누르는 '처형' 한 방이 15~20분 플레이 동안 스킬 빌드로 점점 강해질 수 있는지 —
본 개발로 넘어가기 위한 단 하나의 관문을 확인한다.

기반 버전 · v0.3.0 · 순간 베기 처형 / 늦은 죽음
날짜 · 2026-06-30
형식 · 토론 아젠다 → 세부 → 결론
01 · 토론 아젠다

본 개발로 갈지 정하는 핵심 질문

"플레이어가 직접 누르는 처형이, 15~20분 플레이 동안
스킬 빌드로 점점 강해지는 성장이 가능한가?"

그렇다 → 본 개발 시작  ·  아니다 → 핵심 설계 다시 짜기. 아래 따져볼 점 3개는 따로 노는 안건이 아니라 모두 이 질문에 딸린 질문이다.

갈래 ①

ZZZ식 늦은 죽음

체력 0이어도 바로 안 죽음. 처형 연출이 끝나야 죽음. 기절 게이지 → 비틀대기 → 마무리 일격.

갈래 ②

처형을 꼭 해야 하나

정예마다 직접 처형을 꼭 하게 만들지, 잘하는 사람의 선택으로 둘지.

갈래 ③

강화 카드 갈래 나누기

단순 능력치 카드를 성격별로 갈라 20분 성장 곡선을 만들 수 있는가.

02 · 세부 — 핵심 주장

자동 무기 키우기 ≠ 직접 누르는 처형 키우기

빌드 투자는 누르는 횟수가 아니라
한 번 누를 때의 위력을 키워야 한다.

실패 더 자주 누르는 빌드

처형 쿨타임이 짧아져 18분엔 처형 연타 → 손목 부상 + 연출 지겨움. 사람 손과 연출 감상엔 한계가 있다.

성공 더 세게 터지는 빌드

18분엔 덜 누르지만, 한 번 눌러 연속 처형 12회로 화면을 통째로 쓸어버리는 한 방이 된다.

⮕ 이 오른쪽 위로 솟는 곡선이 느껴지면 관문 통과. 안 그려지면 핵심 설계 다시.

03 · 세부 ① — ZZZ식 기절 장치

①늦은 죽음 = 빌드를 굴리는 연료통

  • 지금: 비틀대기(그로기)는 정예에게만 + 자동 베기로 체력 0을 만들 때만 생김 → 처형 연료가 부족.
  • 제안: 비틀대기를 모든 적의 기절 게이지로 넓히기(ZZZ 방식). 게이지가 차면 → 비틀대기(죽음 대기).
  • 처형 키 한 번으로 대기 중인 적 전부를 연속 처형 → "모았다 한 방"이 규칙으로 자리 잡음.
  • 늦은 죽음 = 연속 처형을 모으는 대기 시간. 단순 멋이 아니라 빌드의 연료통이라는 이유가 생김.

⚠ 화면 완전 멈춤은 관문의 적

18분엔 처형이 잦아지는데 매번 화면이 완전히 멈추면 게임이 뚝뚝 끊긴다.

→ 완전 멈춤 대신 크게 느려지기(0.1~0.2배) + 짧은 무적. 멋은 살리고 · 무한 멈춤 악용은 막기.

04 · 세부 ② — 처형을 꼭 해야 하나

②'억지로 시키기'가 아니라 '하고 싶게 만들기'로

15~20분 플레이에서 "정예마다 직접 처형을 꼭 하게" 만들면 빌드 키우기와 정면으로 부딪힌다.
18분에 기절한 적 40마리를 하나씩 누르면 강해지는 손맛이 아니라 노동이다.

억지로 ❌

안 누르면 안 죽음 → 잔손질 강요 → 손목 부상.

끌리게 ✅

그냥 둔 적: 죽긴 죽되 보상 적음(지금 faded 처리와 같은 생각).
처형한 적: 빌드로 커진 위력(넓은 폭발·연속 기절·회복·가속 강화)이 터짐.

꼭 누르게 만드는 힘은 규칙(안 누르면 안 죽음)이 아니라
이득(누르면 내 가장 센 수단이 터짐)에서 나온다.
05 · 세부 ③ — 강화 카드 갈래 나누기

③한 방의 위력을 키우는 세 갈래 + 진화

갈래성격빌드가 키우는 것곡선에 주는 도움
처형 빌드
한 방형		모았다 한 방처형 범위 · 연속 처형 최대 수 · 비틀대기 시간 · 처형 시 넓은 폭발위력 ↑↑ (핵심)
휩쓸기 빌드
자동 정리형		기절 자동으로 뿌리기베기 수 · 공격 속도 · 사정거리 + 베기가 기절 게이지 쌓음처형 연료 공급 ↑
몰아치기 빌드
속도형		박자 · 눈덩이 효과오버드라이브(가속) 중첩/지속 · 처형마다 강화 · 피 흡수 · 생존처형 → 다음 처형 빨라짐
★ 진화 · 화면 전체 처형

처형 범위 최대 + 연속 처형 최대 → 화면에 비틀대는 적 전부를 한꺼번에 처형.

★ 진화 · 자동 기절 장판

베기 최대 + 기절 최대 → 근처 잡몹이 알아서 처형 대기 상태가 됨. 자동 무기가 처형 연료를 계속 공급 = 뱀서다운 눈덩이 효과 되살리기.

06 · 결론 — 먼저 확인할 것

본 개발로 넘어가기 전, 증명할 단 하나

"같은 처형 키인데, 2분엔 1마리 / 18분엔 12마리 + 화면 청소로
느껴지는가?" — 이 곡선 하나만 느껴지면 관문 통과.
1
기절 게이지를 모든 적으로
정예 전용 비틀대기 → 잡몹까지. 자동 베기가 기절 게이지 쌓음.
2
쿨타임 환급 없애기 → 위력으로
연속 처형 수에 넓은 폭발·기절 퍼뜨리기 보상을 몰아줌.
3
화면 완전 멈춤 → 느린 화면
무한 멈춤 악용 막고, 멋은 살리기.
4
세 갈래 카드 + 진화 1개
화면 전체 처형 또는 자동 기절 장판 + 적 강해지는 구간 18분까지 늘리기.
07 · 토론 계속 — 미해결 쟁점

다음 갈림길 2개

쟁점 A

쿨타임 환급을 완전히 없앨까?

지금은 "한 마리 잡을 때마다 쿨타임 즉시 초기화"라 손맛은 있지만, 이게 성장 곡선을 평평하게 만드는 주범 → 없애자는 입장. 즉시 초기화되는 손맛에 미련이 있는가?

쟁점 B

기절을 잡몹까지 넓힐까?

넓히는 순간 무게추가 뱀서(자동으로 쓸기) → ZZZ식 액션(직접 처형)으로 넘어간다. 이 성격 변화를 일부러 노리는가, 아니면 기절은 정예에게만 두고 뱀서다움을 지킬 것인가?

⮕ A·B 입장이 정해지면 세 갈래 카드 전체 설계 또는 먼저 확인할 검증 코드로 진행.
